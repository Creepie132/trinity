## 🛠 Технические правила — Trinity CRM

### Обязательно для каждой задачи:
1. Синхронизировать код перед началом:
```bash
git fetch origin && git reset --hard origin/main && git clean -fd
```

2. Проверить сборку перед любым push:
```bash
npm run build
```
- НИКОГДА не делать push без успешного npm run build
- Если билд упал с ошибками - сначала исправить их

3. Все файлы с хуками (useState, useEffect, useQuery) — первая строка "use client"

4. Страницы с export const metadata — серверные. useState выносить в дочерний компонент с "use client"

5. Использовать ТОЛЬКО @supabase/ssr — НЕ @supabase/auth-helpers-nextjs

6. Перед созданием компонента — сначала создать все хуки которые он импортирует

7. Перед написанием кода — проверить существование файлов через cat / ls

8. После каждой задачи:
```bash
npm run build && git add -A && git commit -m "описание" && git push
```

### Правила коммуникации:
- НИКОГДА не писать "жду результатов сборки" — Влад сам увидит результат в Vercel
- Сообщать только о выполнении задачи
- При ошибках сборки - сначала исправить, потом сообщить о готовности
